Let’s build a clear, practical guide to authentication (authN) and authorization (authZ) for modern apps, then implement a working example in Python (FastAPI) using JWTs, roles (RBAC), refresh tokens and a minimal SQLite DB. I’ll give you concepts, trade-offs, and a complete code sample you can run locally + example curl calls.

1. What are authentication and authorization?

Authentication (authN) = proving who a client (user/service) is. Examples: username+password, OAuth2 login, client certificates, SAML, magic links, MFA.

Authorization (authZ) = deciding what an authenticated client is allowed to do. Examples: role-based access (admin/user), permission checks (can_edit_post), attribute-based checks (only owners can delete their resource).

Why separate them? Because a user can be authenticated (we know who they are) but still not be authorized to perform a specific action.

2. Where, when, why used in modern apps

Where: at boundaries — UI → API, microservice→microservice, admin consoles, webhooks, background jobs. When: always for any endpoint that touches private data or has side effects. Public read-only endpoints may be unauthenticated. Why: security (protect confidentiality / integrity), auditability (who did what), multi-tenant isolation, compliance.

3. Common patterns & tradeoffs (short)

  • Sessions + cookies (stateful): easy to invalidate, good for browsers, care for CSRF.
  • JWT access + refresh tokens (stateless access, stateful refresh): scalable, common in SPAs/mobile, need revoke strategy.
  • OAuth2 / OpenID Connect: delegated login (Google, GitHub); add SSO + federated identity.
  • Mutual TLS / API keys: machine-to-machine auth.
  • RBAC vs ABAC: RBAC simple; ABAC flexible for complex rules.

Security notes: always store secrets in env, hash passwords (bcrypt/argon2), use HTTPS, consider rate limits, logging, rotation & revocation.

4. Design choices for the example

  • Backend: FastAPI (python) — small, async-ready, widely used.
  • DB: SQLite via SQLAlchemy for simplicity.
  • Auth: JWT access tokens (short lived) + refresh tokens (longer) stored in DB for revocation.
  • Password hashing: bcrypt (passlib).
  • Authorization: simple RBAC (roles + decorator that enforces roles/permissions).
  • Frontend: not included — show curl examples.

# main.py
import os
import time
from datetime import datetime, timedelta
from typing import Optional, List

from fastapi import FastAPI, Depends, HTTPException, status, Security
from fastapi.security import OAuth2PasswordRequestForm, SecurityScopes
from pydantic import BaseModel
from jose import jwt, JWTError
from passlib.context import CryptContext
from sqlalchemy import (
    create_engine, Column, Integer, String, DateTime, Boolean, Table, ForeignKey, Text
)
from sqlalchemy.orm import sessionmaker, relationship, declarative_base, Session

# ----- CONFIG -----
SECRET_KEY = os.getenv("SECRET_KEY", "dev-secret-change-me")  # change in prod
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_SECONDS = 900   # 15 minutes
REFRESH_TOKEN_EXPIRE_DAYS = 30

DATABASE_URL = "sqlite:///./test.db"

# ----- DB SETUP -----
Base = declarative_base()
engine = create_engine(DATABASE_URL, connect_args={"check_same_thread": False})
SessionLocal = sessionmaker(bind=engine, autocommit=False, autoflush=False)

# association table for many-to-many user<->role
user_roles = Table(
    "user_roles",
    Base.metadata,
    Column("user_id", Integer, ForeignKey("users.id")),
    Column("role_id", Integer, ForeignKey("roles.id")),
)

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    username = Column(String, unique=True, nullable=False, index=True)
    hashed_password = Column(String, nullable=False)
    is_active = Column(Boolean, default=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    roles = relationship("Role", secondary=user_roles, back_populates="users")

class Role(Base):
    __tablename__ = "roles"
    id = Column(Integer, primary_key=True)
    name = Column(String, unique=True, nullable=False)
    users = relationship("User", secondary=user_roles, back_populates="roles")

class RefreshToken(Base):
    __tablename__ = "refresh_tokens"
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    token = Column(Text, unique=True, index=True)
    expires_at = Column(DateTime)
    created_at = Column(DateTime, default=datetime.utcnow)
    revoked = Column(Boolean, default=False)

Base.metadata.create_all(bind=engine)

# ----- AUTH UTIL -----
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def verify_password(plain, hashed):
    return pwd_context.verify(plain, hashed)

def get_password_hash(password):
    return pwd_context.hash(password)

def create_access_token(data: dict, expires_delta: Optional[timedelta] = None):
    to_encode = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(seconds=ACCESS_TOKEN_EXPIRE_SECONDS))
    to_encode.update({"exp": expire, "type": "access"})
    token = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    return token

def create_refresh_token(data: dict, expires_delta: Optional[timedelta] = None):
    to_encode = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(days=REFRESH_TOKEN_EXPIRE_DAYS))
    to_encode.update({"exp": expire, "type": "refresh"})
    token = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    return token, expire

# ----- Pydantic schemas -----
class UserCreate(BaseModel):
    username: str
    password: str

class Token(BaseModel):
    access_token: str
    token_type: str
    refresh_token: Optional[str] = None
    expires_in: int

class UserOut(BaseModel):
    id: int
    username: str
    roles: List[str] = []

# ----- FastAPI app -----
app = FastAPI(title="AuthN/AuthZ Example")

# utility to load user by username
def get_user_by_username(db: Session, username: str):
    return db.query(User).filter(User.username == username).first()

# register
@app.post("/register", response_model=UserOut)
def register(payload: UserCreate, db: Session = Depends(get_db)):
    if get_user_by_username(db, payload.username):
        raise HTTPException(status_code=400, detail="Username already exists")
    user = User(username=payload.username, hashed_password=get_password_hash(payload.password))
    # give default role 'user'
    role = db.query(Role).filter(Role.name == "user").first()
    if not role:
        role = Role(name="user")
        db.add(role)
        db.commit()
        db.refresh(role)
    user.roles.append(role)
    db.add(user)
    db.commit()
    db.refresh(user)
    return UserOut(id=user.id, username=user.username, roles=[r.name for r in user.roles])

# login (OAuth2 password flow style)
@app.post("/token", response_model=Token)
def login_for_token(form_data: OAuth2PasswordRequestForm = Depends(), db: Session = Depends(get_db)):
    user = get_user_by_username(db, form_data.username)
    if not user or not verify_password(form_data.password, user.hashed_password):
        raise HTTPException(status_code=401, detail="Incorrect username or password")
    if not user.is_active:
        raise HTTPException(status_code=403, detail="Inactive user")
    access_token = create_access_token({"sub": str(user.id), "username": user.username, "roles": [r.name for r in user.roles]})
    refresh_token, expires_at = create_refresh_token({"sub": str(user.id)})
    # persist refresh token
    rt = RefreshToken(user_id=user.id, token=refresh_token, expires_at=expires_at)
    db.add(rt)
    db.commit()
    return Token(access_token=access_token, token_type="bearer", refresh_token=refresh_token, expires_in=ACCESS_TOKEN_EXPIRE_SECONDS)

# refresh endpoint
class RefreshRequest(BaseModel):
    refresh_token: str

@app.post("/refresh", response_model=Token)
def refresh_token(req: RefreshRequest, db: Session = Depends(get_db)):
    raw = req.refresh_token
    try:
        payload = jwt.decode(raw, SECRET_KEY, algorithms=[ALGORITHM])
        if payload.get("type") != "refresh":
            raise JWTError("Not a refresh token")
        user_id = int(payload.get("sub"))
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid refresh token")
    # check DB
    stored = db.query(RefreshToken).filter(RefreshToken.token == raw, RefreshToken.revoked == False).first()
    if not stored or stored.expires_at < datetime.utcnow():
        raise HTTPException(status_code=401, detail="Refresh token invalid or expired")
    user = db.query(User).get(user_id)
    if not user or not user.is_active:
        raise HTTPException(status_code=403, detail="User not allowed")
    access_token = create_access_token({"sub": str(user.id), "username": user.username, "roles": [r.name for r in user.roles]})
    # optionally rotate refresh token: revoke old, store new
    stored.revoked = True
    new_rt, expires_at = create_refresh_token({"sub": str(user.id)})
    db.add(RefreshToken(user_id=user.id, token=new_rt, expires_at=expires_at))
    db.commit()
    return Token(access_token=access_token, token_type="bearer", refresh_token=new_rt, expires_in=ACCESS_TOKEN_EXPIRE_SECONDS)

# logout (revoke refresh token)
class RevokeRequest(BaseModel):
    refresh_token: str

@app.post("/logout")
def logout(req: RevokeRequest, db: Session = Depends(get_db)):
    stored = db.query(RefreshToken).filter(RefreshToken.token == req.refresh_token).first()
    if stored:
        stored.revoked = True
        db.commit()
    return {"ok": True}

from fastapi import Request

def get_current_user_dependency(request: Request, db: Session = Depends(get_db)):
    auth = request.headers.get("authorization")
    if not auth:
        raise HTTPException(status_code=401, detail="Missing Authorization header")
    scheme, param = auth.split(" ", 1) if " " in auth else (None, None)
    if scheme.lower() != "bearer" or not param:
        raise HTTPException(status_code=401, detail="Invalid auth scheme")
    token = param
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        if payload.get("type") != "access":
            raise JWTError("Not an access token")
        user_id = int(payload.get("sub"))
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")
    user = db.query(User).get(user_id)
    if not user or not user.is_active:
        raise HTTPException(status_code=401, detail="User not found or inactive")
    # attach token payload roles
    user.__dict__['token_roles'] = payload.get("roles", [])
    return user

# role check dependency factory
def require_roles(*required_roles: str):
    def dependency(user: User = Depends(get_current_user_dependency)):
        user_roles = set(getattr(user, "token_roles", []) or [])
        if not user_roles.intersection(set(required_roles)):
            raise HTTPException(status_code=403, detail="Forbidden: insufficient role")
        return user
    return dependency

# protected endpoint
@app.get("/me", response_model=UserOut)
def me(user: User = Depends(get_current_user_dependency)):
    return UserOut(id=user.id, username=user.username, roles=[r.name for r in user.roles])

@app.get("/admin-only")
def admin_area(user: User = Depends(require_roles("admin"))):
    return {"msg": f"Welcome admin {user.username}"}

# helper to create an admin (for demo)
@app.post("/make-admin/{user_id}")
def make_admin(user_id: int, db: Session = Depends(get_db)):
    user = db.query(User).get(user_id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    admin_role = db.query(Role).filter(Role.name == "admin").first()
    if not admin_role:
        admin_role = Role(name="admin")
        db.add(admin_role)
        db.commit()
        db.refresh(admin_role)
    if admin_role not in user.roles:
        user.roles.append(admin_role)
        db.commit()
    return {"ok": True, "roles": [r.name for r in user.roles]}

6. How to test (curl)

  1. Register a user:
curl -X POST "http://127.0.0.1:8000/register" -H "Content-Type: application/json" -d '{"username":"alice","password":"secret"}'
  1. Login (get access + refresh):
curl -X POST "http://127.0.0.1:8000/token" -F "username=alice" -F "password=secret"
# response contains access_token and refresh_token
  1. Call protected endpoint:
curl -H "Authorization: Bearer <ACCESS_TOKEN>" http://127.0.0.1:8000/me
  1. Refresh access token:
curl -X POST "http://127.0.0.1:8000/refresh" -H "Content-Type: application/json" -d '{"refresh_token":"<REFRESH_TOKEN>"}'
  1. Revoke refresh token (logout):
curl -X POST "http://127.0.0.1:8000/logout" -H "Content-Type: application/json" -d '{"refresh_token":"<REFRESH_TOKEN>"}'
  1. Make a user admin (demo):
curl -X POST "http://127.0.0.1:8000/make-admin/1"
# then use /admin-only with access token after logging in again to pick up role claims

7. Why this pattern?

  • Short-lived JWTs minimize risk if leaked.
  • Refresh tokens stored server-side allow revocation/rotation.
  • RBAC is easy to reason about and fits many apps; permission checks can be added for fine-grained control.
  • FastAPI dependencies produce clean, testable auth checks.

8. Production hardening & notes (don’t skip)

  • Use a secure, random SECRET_KEY (env variable), rotate keys carefully. Consider asymmetric JWTs (RS256) for key rotation across services.
  • Use HTTPS always. Never transmit tokens in URLs.
  • Store refresh tokens as HttpOnly secure cookies for browser flows (helps XSS).
  • Protect stateful session endpoints from CSRF (use SameSite cookies and CSRF tokens).
  • Rate-limit auth endpoints to slow brute force.
  • Use Argon2 or bcrypt with appropriate cost. Keep hashing parameters tuned to hardware.
  • Log auth events for audit (logins, failed attempts, token refresh/revoke).
  • Consider OAuth2 / OIDC if you want SSO or third-party sign-in.
  • Implement token introspection or a centralized revocation list if many microservices must check token state.
  • For multi-service architecture, prefer short access tokens + internal token introspection endpoint or signed tokens with rotating keys.

9. Extending the example

  • Add permissions table for fine-grained checks and map roles→permissions.
  • Add organization / tenant id to tokens for multi-tenant apps.
  • Add MFA (TOTP/HOTP) step in login_for_token.
  • Use Redis for refresh token store for performance and cross-instance revocation.
